Yandex Tiles Demo

Демо для проверки, как тайлы Яндекса ведут себя в стороннем движке карты. Основа рендера здесь — Leaflet, а тайлы берутся с tiles.api-maps.yandex.ru.

Практическая оговорка: Leaflet использует Web Mercator. Поэтому projection=web_mercator здесь корректный базовый режим. wgs84_mercator оставлен для экспериментов с raw tiles, но может визуально не совпадать.

Показывать сетку тайлов
Удобно для проверки x/y/z в центре и на краях.
Не сбрасывать текущее положение при Apply
Если выключить, будет использован lat/lon/zoom из формы.
Leaflet работает в Web Mercator. При wgs84_mercator вы смотрите скорее на совместимость raw tiles, чем на строго корректную геометрию.

Если браузер странно ведет себя с локальным файлом, поднимите простой сервер, например python3 -m http.server, и откройте эту страницу по http://localhost:8000/....

Важно: endpoint тайлов Яндекса не выглядит как CORS-enabled для браузерного чтения пикселей. Поэтому здесь тайлы грузятся как обычные изображения. Смотреть карту можно, но пытаться читать пиксели тайлов через canvas без прокси нельзя.

Технически это демонстрирует, что тайлы можно подставить в другой map engine. Отдельно нужно проверить лицензионные ограничения Яндекса на такой сценарий.
